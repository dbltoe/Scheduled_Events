v2.0.0
- Rebuilt as an encapsulated plugin, replacing the older non-encapsulated version
  of Scheduled Events. Requires Zen Cart 2.0.0 or later; not compatible with
  pre-2.0.0 installations. Developed and tested against Zen Cart v2.2.2/v2.2.3.
- Storefront Scheduled Events page (main_page=events): lists every event whose
  display window currently qualifies, each as a name (H2) followed by place,
  start/stop dates, and optional comments, booth location, event information,
  and driving directions in a blockquote. A separator <hr> appears between
  consecutive events (none shows with zero or one event listed). Shows a
  configurable "no events scheduled" message, naming the current date window,
  when nothing qualifies.
- Booth Location (required) and Event Information (optional) are each a
  plain-text description with a separate, optional URL; the description
  becomes a link only when a valid URL is given, otherwise it displays as
  plain text - no HTML authoring required from the store owner. The URL
  accepts a full external address or a root-relative path to an asset
  already on the store (e.g. /images/mapName.jpg), so a locally-hosted
  map image doesn't require an absolute URL.
- Driving Directions accepts a plain address/URL (opens Google Maps in a new
  tab) or a Google Maps "Embed a map" <iframe> code (opens as a self-contained,
  on-page pop-up map instead - no page navigation, full-screen on phones/
  tablets, closed via an X button, clicking outside, or Escape). The embedded
  map is lazy-loaded and never fetched unless the pop-up is actually opened.
- Sidebox Display Mode (Configuration > Scheduled Events) chooses how upcoming
  events are promoted: "Information Listing" adds a link into the current
  template's existing Information sidebox (styled to match responsive_classic
  and ZCA Bootstrap out of the box); "Bootstrap Sidebox" instead shows the
  plugin's own auto-scrolling carousel box, placed via Design > Layout Boxes
  Controller (requires a Bootstrap-based template). A master "Enable
  Scheduled Events Display" configuration switch can fully hide the
  storefront page and both sidebox modes at any time, without uninstalling.
- Admin event management screen (Catalog > Scheduled Events): add/edit/delete
  events, with required-field markers, a sort-order dropdown for the event
  list, and a Help button linking to this plugin's GitHub repository.
- Active field: each event can be paused (unchecked) without deleting it or
  changing its dates, hiding it from the storefront page and both sidebox
  modes. The admin Catalog list shows active events only by default, with a
  "Show All" link to reveal paused events (shown dimmed, with a Status
  column) for editing or reactivation.
- Admin configuration group (Configuration > Scheduled Events) for the page
  heading, all field labels, the no-events message (and its follow-up line),
  the 30/60/90-day display window, the sidebox display mode/title/max items,
  and the master enable/disable switch.
