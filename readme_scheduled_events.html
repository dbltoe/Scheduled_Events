Scheduled Events for Zen Cart

Version 2.0.0 — requires Zen Cart v2.0.0 or later (developed and tested against v2.2.2/v2.2.3). Author: dbltoe.

Overview

Scheduled Events lets a store owner maintain a list of upcoming events (shows, fairs, expos, appearances, etc.) and display them on their own storefront page. Events are managed from Catalog > Scheduled Events in the admin, and a link to the events page is added into the current template's existing Information sidebox. That link's styling is set to match responsive_classic and ZCA Bootstrap out of the box; using any other template may require adjusting its styling to match that template's own conventions.

The storefront page heading, all field labels, the "no events scheduled" message, the display time window (how many days before an event's start date it becomes visible), and the sidebox link text are all configurable under Configuration > Scheduled Events. A master "Enable Scheduled Events Display" switch there can fully hide the storefront page and sidebox link at any time, without uninstalling the plugin.

Event Fields

Each event has the following fields, set from Catalog > Scheduled Events:

  • Name, Place, Start Date, Stop Date — required for every event.
  • Comments — optional free text.
  • Booth Location (required) and Event Information (optional) — each is a plain-text description with a separate, optional URL field. If a valid URL is given, the description becomes a link (opened in a new tab); if the URL is missing or doesn't validate, it's silently ignored and the description just displays as plain text. The store owner never has to author HTML.
  • Driving Directions — accepts either a plain address or URL (the storefront link opens Google Maps in a new tab), or a Google Maps "Embed a map" <iframe> code (the storefront link instead opens the map as a self-contained pop-up over the page — no page navigation, full-screen automatically on phones/tablets, closed via its X button, clicking outside the pop-up, or the Escape key). The embedded map is lazy-loaded: it is never fetched unless a visitor actually opens the pop-up.

On the storefront, events are listed in order, each as its name (as an H2) followed by its details in a blockquote; a horizontal rule separates consecutive events (none appears with zero or one event listed). When no events currently qualify for display, the page instead shows a configurable message naming the current date window, followed by a "check back later" line.

Installing, Upgrading, or Uninstalling

Initial Installation or Upgrade

Copy the contents of the zc_plugins/ScheduledEvents/v2.0.0 folder into your Zen Cart store's root directory, preserving the folder structure. Then, in the admin, go to Tools > Install/Update Plugins, locate Scheduled Events, and click Install (or Update, if upgrading from a prior version).

This creates the eventz database table (respecting your store's table prefix, if any), adds the Configuration > Scheduled Events settings group, and adds the Catalog > Scheduled Events admin menu entry.

If Sidebox Display Mode (Configuration > Scheduled Events) is set to "Bootstrap Sidebox", the box will need to be (re-)placed in a column via Design > Layout Boxes Controller after installing - its placement does not survive an uninstall (see below), so this is also needed again after any uninstall/reinstall cycle.

Uninstalling

In the admin, go to Tools > Install/Update Plugins, locate Scheduled Events, and click Uninstall. This removes the eventz table and all of its data, the plugin's configuration settings, the admin menu entry, and the "Bootstrap Sidebox" box's placement/position in Design > Layout Boxes Controller, if it had been placed in a column. Removing the plugin's files from the server afterward is optional but recommended.
